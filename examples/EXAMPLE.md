# 使用示例

## 快速开始示例

### 示例 1：创建一个基础的领导skill（无需数据采集）

```
用户：帮我创建一个领导skill
AI：好的，我们来创建一个领导skill。请告诉我：

1. 花名/代号（必填）：
2. 基本信息（一句话）：公司、职级、职位、性别、年龄
3. 领导风格标签（一句话）：MBTI、管理风格、核心特质
4. 你与他的关系（一句话）：

用户：
1. 花名：张总
2. 基本信息：字节 2-2 技术总监 男 35岁
3. 领导风格标签：ENTJ 狮子座 控制欲强 结果导向
4. 我的关系：我是他下属，想知道怎么跟他相处

AI：好的，我来汇总一下信息...确认生成吗？

用户：确认

AI：✅ 领导 Skill 已创建！
    文件位置：leaders/zhang-zong/
    触发词：
      /zhang-zong（完整版）
      /zhang-zong-leadership（仅领导力模拟）
      /zhang-zong-upward（仅向上管理）
      /zhang-zong-replacement（仅取代路径规划）
```

### 示例 2：使用领导力模拟

```
用户：/zhang-zong 如果现在项目延期了，张总会怎么处理？

AI：[基于张总的领导风格模拟]
根据张总的领导风格（ENTJ 狮子座 控制欲强 结果导向），他可能会：

1. **第一时间问责**：立刻找到负责人，问清楚为什么延期
2. **只看结果**：不听任何借口，只要解决方案
3. **强势施压**：要求必须在某个时间点前完成，不计代价
4. **自己掌控**：可能会亲自接管项目，全程监控

你可以这样应对：
- 主动提前汇报，不要等他来问
- 直接给出解决方案，不要找借口
- 展现你能掌控局面，让他放心
```

### 示例 3：使用向上管理

```
用户：/zhang-zong-upward 我有一个新方案想给张总汇报，怎么说他才会听？

AI：[基于张总的向上管理分析]
根据对张总的分析，给他汇报时要注意：

✅ **汇报技巧**：
- 开门见山，直接说结论，不要铺垫太多
- 用数据说话，给他看具体的数字和指标
- 强调结果和收益，不要只讲过程

✅ **说服方式**：
- 告诉他"这个方案能帮你达成XX目标"
- 用"你肯定会关心..."这样的句式
- 如果可能，找一个他信任的人帮你背书

✅ **雷区规避**：
- ❌ 不要说"我觉得..."，要说"数据显示..."
- ❌ 不要挑战他的权威，即使你是对的
- ❌ 不要在公开场合质疑他

汇报模板：
"张总，关于XX项目，我有一个方案，能帮我们把效率提升30%。核心思路是..."
```

### 示例 4：使用取代路径规划

```
用户：/zhang-zong-replacement 我想知道怎么才能取代张总这个位置？

AI：[基于张总的取代路径规划分析]
⚠️ 伦理提醒：以下分析仅供个人成长参考，建议通过正当途径获得职业发展。

## 取代路径规划：张总

### 一、能力缺口分析
- 他的核心能力：技术决策能力、向上管理能力、资源整合能力
- 你的差距：技术深度可能还不够，向上管理经验不足
- 学习路径：1）深入学习他负责的技术栈 2）观察他怎么跟上级沟通

### 二、软肋分析
- 他的致命软肋：脾气太差，团队凝聚力一般
- 潜在风险点：对细节关注不够，容易出小问题

### 三、晋升路径分析
- 他是怎么上来的：抓住了上一个技术负责人离职的机会
- 这个位置需要什么：技术能力 + 向上管理能力 + 能扛事

### 四、你的策略建议

**短期（3-6个月）**：
- 重点表现：技术能力 + 团队凝聚力
- 避免什么：不要跟他正面冲突
- 搞好关系：跟他的上级建立良好印象

**中期（6-12个月）**：
- 掌握什么：他负责的核心系统
- 做出什么：一个能让你脱颖而出的项目
- 建立人脉：跟其他部门负责人搞好关系

**长期（1-2年）**：
- 怎么布局：在团队中建立自己的影响力
- 机会出现时：展现你能比他做得更好
- 万一失败：积累的能力也能让你去其他团队
```
