# 对话纠正处理指南

## 纠正识别

当用户表达以下内容时，进入纠正模式：
- "不对" / "这不对"
- "他不会这样" / "他不是这样的"
- "他应该是" / "其实他是"
- "更正一下" / "修正一下"

---

## Step 1：识别纠正内容

分析用户的纠正，判断属于哪个类别：

### 类别 A：Leadership（领导风格）
- 决策模式相关
- 沟通风格相关
- 用人偏好相关
- 风险承受力相关

### 类别 B：Upward（向上管理）
- 汇报偏好相关
- 说服方式相关
- 雷区规避相关
- 需求预判相关

### 类别 C：Replacement（取代路径）
- 能力缺口相关
- 软肋分析相关
- 晋升路径相关
- 竞争对手分析相关

---

## Step 2：生成 correction 记录

为每次纠正生成一条记录，包含：
- 纠正时间
- 纠正内容
- 原内容摘要
- 新内容说明

记录格式：
```markdown
## Correction 记录 - {日期}

### 原内容
{原内容的简要摘要}

### 用户纠正
{用户的原话}

### 更新内容
{更新后的内容}
```

---

## Step 3：追加到对应文件

用 Edit 工具追加到对应文件的 `## Correction 记录` 节。

如果文件中还没有 `## Correction 记录` 节，在文件末尾添加：

```markdown
---

## Correction 记录

{在此处追加纠正记录}
```

---

## Step 4：更新主内容

除了追加纠正记录，还要更新文件的主内容：

### 如果是补充信息
在相关章节附近添加补充内容。

### 如果是修正错误
用以下方式标注：
```markdown
> [更正 {日期}] {正确的内容}
>
> 原内容：{原内容}
```

或者，如果原内容完全错误，直接替换并标注。

---

## Step 5：重新生成 SKILL.md

纠正完成后，重新生成完整的 SKILL.md：

1. 读取更新后的三个文件（leadership.md、upward.md、replacement.md）
2. 读取 meta.json
3. 按照 SKILL.md 结构重新生成
4. 用 Write 工具写入 leaders/{slug}/SKILL.md

---

## Step 6：更新 meta.json

更新 meta.json 的以下字段：
- `updated_at` - 设置为当前时间
- `version` - 版本号递增（v1 → v2 → v3...）
- `corrections_count` - 递增 1

---

## 纠正原则

1. **尊重用户**：用户最了解这位领导，用户的纠正是权威的
2. **保留历史**：不要删除原内容，而是记录纠正过程
3. **及时更新**：主内容要反映最新的认知
4. **版本备份**：每次纠正前都备份当前版本
